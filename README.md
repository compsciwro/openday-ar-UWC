A playful **augmented reality experience** created for the **University of the Western Cape Open Day**.
This project combines a sweet pastel theme, mobile interaction, and AR image tracking to create a fun and memorable experience for visitors.

Users are welcomed with a custom candy-inspired start screen, then guided to scan a themed image target. Once scanned, an adorable 3D **Bee Macaron** character appears in augmented reality.

---

## 🌟 Project Overview

The **Open Day AR Experience** was designed to make the Open Day event more interactive and engaging. Instead of only viewing static information, users can interact with a creative AR experience directly from their mobile device.

The project uses a sweet-themed visual style with soft colours, candy graphics, and a cute 3D character reveal to create a friendly and exciting first impression.

---

## 📱 How It Works

1. The user opens the AR experience.
2. A themed start screen introduces the activity.
3. The user taps **Start Scanning**.
4. The camera opens and scans the sweet-themed AR target.
5. A 3D **Bee Macaron** character appears on top of the image target.

## 🛠️ Technologies Used

* **Unity** — main development platform
* **C#** — scripting and interaction logic
* **AR image tracking** — used to detect the target image
* **3D assets** — used for the AR character
* **Adobe Photoshop / Design Tools** — used for visual design and themed graphics

---

## 🎯 Purpose of the Project

The goal of this project was to create an interactive AR activity that could be used during an Open Day setting. It was designed to show how augmented reality can make events more exciting, visual, and memorable.

The experience also demonstrates how AR can be used beyond entertainment, especially in education, exhibitions, campus events, and interactive demonstrations.

---

## 🚀 Getting Started

To run the project locally:

1. Clone the repository:

```bash
git clone https://github.com/your-username/open-day-ar-experience.git
```

2. Open the project in **Unity**.

3. Make sure the required AR packages and image target setup are included.

4. Build and run the project on a supported mobile device.

---

## 📌 Notes

* This project was created as a themed AR prototype for an Open Day experience.
* The AR character appears when the correct image target is scanned.
* The project is best viewed on a mobile device with camera access.
* The visuals were designed to match a fun, soft, candy-inspired theme.
* This project uses Vuforia Engine for AR image tracking.  
    The Vuforia package file is not included in the repository because it exceeds GitHub’s file size limit.  
    To run the project, install Vuforia Engine in Unity and make sure the image target setup is configured.
